## 룰렛 시각 정렬 버그 수정 (지급 오류 아님)

### 분석 요약
- **서버 지급 로직(`spin_roulette` RPC)은 100% 정상.** 세그먼트 인덱스 → 금액 매핑이 일관되며, 클라이언트는 금액을 보내지 않고 서버 결과만 표시한다 (Stack Overflow 베스트 패턴 준수 중).
- **시각 버그**: `Roulette.tsx`의 라벨 배치 각도가 **conic-gradient의 시작 기준(12시)** 과 **CSS rotate 기준(3시)** 사이의 90° 차이를 보정하지 않음 → 색상 영역 위에 다른 세그먼트의 라벨이 표시됨.
- 결과: 포인터(12시)가 가리키는 색상은 서버가 반환한 정답(예: 5,000원 영역)이지만, 그 위에 잘못된 라벨("1,000원")이 보여 사용자가 "지급 오류"로 오인.

### 수정 사항 (`src/pages/Roulette.tsx`)

1. **라벨 각도 90° 보정**
   ```ts
   const angle = i * SEG_DEG + SEG_DEG / 2 - 90;
   ```
   - conic-gradient의 12시(0deg) 시작과 동일한 좌표계로 통일.

2. **라벨 텍스트 회전 보정**
   - 현재 `transform: rotate(90deg)` (고정)으로 글자가 항상 같은 방향을 보게 하려 했으나, 90° 시프트 후 일부 라벨이 거꾸로 읽힘.
   - 각 라벨이 중심을 향한 자연스러운 방향이 되도록 `rotate(90deg)` 유지하되, translateX 후 한 번 더 보정 (필요 시 위/아래 절반에서 추가 180° 플립으로 가독성 확보).

3. **세그먼트-라벨 정합성 시각 검증용 데이터 속성 추가**
   ```tsx
   <div data-seg={i} data-label={s.label} ...>
   ```
   - 추후 디버깅 시 DOM에서 즉시 확인 가능.

4. **결과 강조 UI 보강 (오인 방지)**
   - 스핀 종료 후 당첨된 세그먼트를 ring/glow로 시각적 하이라이트해 "포인터가 가리키는 칸 = 결과 금액"임을 명확히 표시.
   - 예: `lastResult.segment`에 해당하는 라벨에 `text-gold animate-pulse` 적용.

5. **결과 카드 강화**
   - "결과 / 금액" 외에 "지급 검증 완료 ✓" 마이크로카피 + 거래 내역 링크를 작은 텍스트로 추가해 사용자 신뢰 회복.
